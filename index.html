Security Wind

Project Draft PDF

Fast • Simple • Light Weight

Abstract:

With an estimated 3.5 billion users of smartphones (techjury 2022), the attack surface of malware infecting mobiles phone is higher than ever, and of the total smartphone users over 2.8 billion of them are users of Android smartphones (Curry 2022). In 2019 malware researchers from GData found more than 4.18 million malicious apps and packages targeting Android phones which were available across multiple application stores and peer to peer transmission methods (GData 2019). Due to the number of users and the amount malware they are susceptible to, we looked to create a way for users to be able to recognize between harmless and a malicious application on the Google Play Store. As applications can only function based on the permissions they are given, we focused on how we can differentiate malware from benign based on the permissions an application asks for. Having a simple way for users to quickly and easily find out whether an application they want to download is malicious is a driving factor is this research. We will do this by creating a trained machine learning algorithm that will provide an inference on the maliciousness of different applications by utilizing datasets of permissions belonging to both benign and malicious Android applications. With this testing our model will be able to differentiate between benign and malicious applications on the Google Play Store. Utilizing a scalable architecture through Microsoft Azure cloud services the solution laid out within this paper will have all parts necessary to implement a solution that scales to usage requirements in future works.
